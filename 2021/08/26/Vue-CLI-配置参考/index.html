Vue CLI(配置参考)

全局CLI配置

有些针对 @vue/cli 的全局配置，例如你惯用的包管理器和你本地保存的 preset，都保存在 home 目录下一个名叫 .vuerc 的 JSON 文件。

vue.config.js

可选的配置文件，若项目的（和package.json同级）根目录中存在这个文件，则会被自动加载。
你可以使用package.json中的vue字段，但是要严格按照JSON的格式来写
这个文件应导出一个包含了选项的对象

publicPath

打包bulid完后，都单独生成一个/dist文件夹
publicPath设置的是域名根目录到项目根目录间的url，默认’/‘是根目录，部署应用包的基本URL，不是项目打包出来的文件存放的位置

outputDir

用来设置项目打包生成的文件的存储目录，可以是静态路径也可以是相对路径
相对路径是相对于文件当前的根路径

assetsDir

用来设置放置打包生成的静态资源（js、css、img、fonts）的目录
该目录相对于outputDir

indexPath

用于设定打包生成的 index.html 文件的存储位置
该路径若是相对路径，则相对于outputDir，也可以是一个绝对路径；
路径一定要以文件名+后缀结尾，最好以index,html结尾。

filenameHashing

设置打包生成的的静态资源的文件名中是否加入hash以便控制浏览器缓存问题。
当只需要保存静态资源时，可以设置filenameHashing为false

代理 devServer

解决开发环境的跨域问题
（跨域就是不同源，即不满足协议、域名、端口都相同的约定）
（同源是指：协议、域名、端口都相同）
假设要把http://localhost:8081/allin/policy/getProductInfo 中的域名换成 www.test.com 相当于把请求换成 http://www.test.com/allin/policy/getProductInfo
代理：在本地会创建一个虚拟服务端，然后发送请求的数据，并同时接收请求的数据，这样客户端端和服务端进行数据的交互就不会有跨域问题
