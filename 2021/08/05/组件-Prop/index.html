组件-Prop

通过Prop向子组件传递数据

Prop是我们可以在组件上注册的一些自定义attribute，一个组件默认可以拥有任意数量的prop，无论任何值都可以传递给prop
使用v-bind可以动态传递prop

Prop类型

若希望每个prop都有指定的值类型，可以以对象形式列出prop，这些property的名称和值分别是prop各自的名称和类型
这不仅可以为组件提供文档，还会在在遇到错误的类型时从浏览器的javaScript控制台提示用户

传递静态或动态的Prop

prop可以给他传入一个静态的值，也可以通过v-bind或简写:动态赋值
任何类型的值都可以传给一个prop

单向数据流

所有的prop都使得其父子之间形成了一个单向下行绑定：父级prop的更新
会向下流动到子组件中，但是反过来则不行。这样会防止从子组件意外变更父级组件的状态，从而导致应用的数据流难以理解
另外，每次父级组件发生变更时，子组件中所有的 prop 都将会刷新为最新的值。这意味着你不应该在一个子组件内部改变 prop。如果这样做了，Vue 会在浏览器的控制台中发出警告。

两种常见的试图变更一个prop的情形

  1. 这个prop用来传递一个初始值；这个子组件接下来希望将其作为一个本地的prop数据来使用。在这种情况下，最好定义一个本地的data property并将这个prop作为其初始值：
  2. 这个prop以一种原始的值传入且需要进行转换。在这种情况下，最好使用这个prop的值来定义一个计算属性 提示：注意在 JavaScript 中对象和数组是通过引用传入的，所以对于一个数组或对象类型的 prop 来说，在子组件中改变变更这个对象或数组本身将会影响到父组件的状态。

Prop验证

prop的验证方式：为props中的值提供一个带有验证需求的对象，而不是一个字符串数组

提示：注意那些 prop 会在一个组件实例创建之前进行验证，所以实例的 property (如 data、computed 等) 在 default 或 validator 函数中是不可用的。

类型检查

type可以是下列原生构造函数中的一个：
String Number Boolean Array Object Date Function Symbol
此外type还可以是自定义构造函数，且通过instanceof来检查确认。

Prop 的大小写命名 (camelCase vs kebab-case)

当你使用 DOM 中的模板时，camelCase (驼峰命名法) 的 prop 名需要使用其等价的 kebab-case (短横线分隔命名) 命名，如果你使用字符串模板，那么这个限制就不存在了。
