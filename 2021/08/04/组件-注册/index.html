组件-注册

组件基础

组件是带有名称的可复用的实例，所以它们可以与根实例接收相同的选项，例如：data、computed、watch、methods以及生命周期钩子等。
在下面这个例子中这个例子是,我们可以把这个组件作为一个根实例中的自定义元素来使用

组件注册

为了能在模版中使用，这些组件必须先注册以便Vue能识别。
注册类型有两种：全局注册和局部注册

组件名

推荐遵循W3C规范中的自定义组件名即（字母全小写且必须包含一个连字符）这可以帮助避免和当前以及未来的HTML元素相冲突
在字符串模版或单个文件组件中定义组件时，定义组件名的方式有两种
1.使用kebab-case 2.使用PascalCase
注意：直接在DOM（即非字符串的模版）中使用只有kebab-case是有效的

全局注册

全局注册的组件可以在应用中的任何组件的模版中使用，在所有的子组件中也是如此。

局部注册

在根组件的components选项中定义使用的组件
注意：局部注册的组件在其子组件中不可用
若希望ComponentA在ComponentB中可用则把ComponentA定义在ComponentB的components选项中

根组件

传递给createApp的选项用于配置根组件。当我们挂载应用时，该组件被用作渲染的起点。

一个应用需要被挂载到一个DOM元素中，例如，如果你想把一个 Vue 应用挂载到

，应该传入 #app

Data Property和方法

Data

组件的data选项是一个函数。Vue在创建新组件实例的过程中调用此函数。它返回的是一个对象，Vue会通过响应性系统将其包裹，并以$data的形式存储在组件实例中。该对象的任何顶级property也直接通过组件实例暴露出来
这些实例property仅在实例首次创建时被添加，所以需确保它们都在data返回的对象中，必要时使用null、undefined占位

方法

用methods选项向组件实例添加方法，它是一个包含所需方法的对象
Vue自动为methods绑定this,以便它始终指向实例，在定义methods时应避免使用箭头函数，因为这会阻止Vue绑定恰当的this指向

计算属性和侦听器

计算属性

对于任何包含响应式数据的复杂逻辑，都应使用计算属性

计算属性的Setter

计算属性VS方法

计算属性是基于它们的响应依赖关系缓存的。
计算属性只在相关响应式依赖发生改变时它们才会重新求值
而每当触发重新渲染时，调用方法将会再次执行函数
如果不希望有缓存，就用method来替代

侦听器

当数据发生变化时，及时作出响应处理
使用watch选项，监听器可以监听数组 / 对象。
允许异步操作 (访问一个 API)，限制我们执行该操作的频率，并在我们得到最终结果前，设置中间状态。这些都是计算属性无法做到的

Class与Style绑定

对象语法

可以传给:class一个对象，以动态的切换class,:class指令可以与普通的class attribute共存

数组语法

把一个数组传给:class，以应用一个class列表
在数组语法中也可以使用对象语法

在组件上使用

在自定义组件上使用class attribute时，这些class将被添加到该元素中。此元素现有的class不会被覆盖

如果你的组件有多个根元素，你需要定义哪些部分将接收这个类。可以使用 $attrs 组件属性执行此操作

绑定内联样式

:style CSS property名可以用驼峰式 (camelCase) 或短横线分隔 (kebab-case，记得用引号括起来) 来命名
同样有对象语法和数组语法
